How Weights Actually Update in SFT

Connecting Logit Gradients to Physical Changes in the Model's Brain

The Problem: Why Update Weights?

In our previous exploration, we found that the gradient of the loss with respect to our logits $\mathbf{z}$ is:

$$\frac{\partial \mathcal{L}}{\partial \mathbf{z}} = \mathbf{p} - \mathbf{y}$$

This tells us what to change: "Increase the score of the correct word, decrease the scores of the wrong words." But logits are just ephemeral numbers generated during a forward pass. They don't persist. To make the model smarter for tomorrow, we must change the permanent structures that created those logits: the weights.

Perspective 1: The Geometry of the Output Layer

How are logits created? Let's look at the very last step of a Large Language Model.

The Transformer has spent 80 layers processing the context and has finally produced a dense vector $\mathbf{h}$ (often called the hidden state or latent representation). This vector represents the model's "thought" about what should come next.

To convert this "thought" into vocabulary scores, we multiply it by the final unembedding matrix $W$:

$$\mathbf{z} = W \mathbf{h}$$
The Geometric Viewpoint:

Think of $W$ not as a grid of numbers, but as a dictionary of vectors. Every row $w_i$ in $W$ is an "embedding vector" for a specific word in the vocabulary. The logit for word $i$ is simply the dot product:

$$z_i = w_i \cdot \mathbf{h}$$

A dot product measures alignment. If the model's thought $\mathbf{h}$ points in the exact same direction as the word vector $w_i$, the score $z_i$ will be massive. If they are orthogonal, the score is zero.

Deriving the Weight Update Step-by-Step

We need to know how changing the matrix $W$ affects the loss $\mathcal{L}$. Using the chain rule from calculus:

$$\frac{\partial \mathcal{L}}{\partial W} = \frac{\partial \mathcal{L}}{\partial \mathbf{z}} \frac{\partial \mathbf{z}}{\partial W}$$

We already know $\frac{\partial \mathcal{L}}{\partial \mathbf{z}} = \mathbf{p} - \mathbf{y}$. Let's denote this error vector as $\delta$.

Because $\mathbf{z} = W \mathbf{h}$, the derivative of $\mathbf{z}$ with respect to $W$ is just $\mathbf{h}^\top$. Thus, the gradient for the matrix $W$ is an outer product:

$$\frac{\partial \mathcal{L}}{\partial W} = \delta \mathbf{h}^\top = (\mathbf{p} - \mathbf{y}) \mathbf{h}^\top$$

Perspective 2: What is this equation actually doing?

Let's look at the update for a single row $w_i$ (the vector representing the $i$-th word in the vocabulary).

The gradient for just this vector is:

$$\frac{\partial \mathcal{L}}{\partial w_i} = (p_i - y_i) \mathbf{h}$$

In Gradient Descent, we update the weights by subtracting the gradient (scaled by a learning rate $\alpha$):

$$w_i \leftarrow w_i - \alpha (p_i - y_i) \mathbf{h}$$

Case A: The Correct Word ($y_c = 1$)

For the correct word, $p_c$ is always less than 1 (e.g., 0.1). So $(p_c - 1)$ is negative (e.g., -0.9).

$$w_c \leftarrow w_c - \alpha (-0.9) \mathbf{h} = w_c + 0.9 \alpha \mathbf{h}$$

Physical Meaning: We take the current "thought" vector $\mathbf{h}$ and add it to the correct word's vector $w_c$. By rotating $w_c$ to be more aligned with $\mathbf{h}$, the dot product $w_c \cdot \mathbf{h}$ will be much larger next time.

Case B: An Incorrect Word ($y_i = 0$)

For an incorrect word, $p_i$ is positive (e.g., 0.2). So $(p_i - 0)$ is positive (e.g., +0.2).

$$w_i \leftarrow w_i - \alpha (+0.2) \mathbf{h} = w_i - 0.2 \alpha \mathbf{h}$$

Physical Meaning: We take the current "thought" vector $\mathbf{h}$ and subtract it from the wrong word's vector $w_i$. We are pushing $w_i$ away from $\mathbf{h}$, ensuring their dot product is smaller next time.

Perspective 3: The Deep Network Backpropagation

Updating the final layer $W$ is only half the battle. What if the problem wasn't the dictionary $W$, but the "thought" $\mathbf{h}$ itself? What if the Transformer generated a stupid thought?

We must also update the parameters deep inside the 80 layers that created $\mathbf{h}$. To do this, we need the gradient of the loss with respect to $\mathbf{h}$:

$$\frac{\partial \mathcal{L}}{\partial \mathbf{h}} = W^\top \frac{\partial \mathcal{L}}{\partial \mathbf{z}} = W^\top (\mathbf{p} - \mathbf{y})$$

What is this physically? It is a weighted sum of the vocabulary vectors. The model looks at the error for every single word, scales the corresponding word vector $w_i$ by its error $(p_i - y_i)$, and adds them all up.

This combined vector tells the lower layers of the neural network: "Hey, your thought $\mathbf{h}$ was wrong. You should have pushed it more in this direction, and less in that direction." This signal then flows backward through all 80 layers (Attention, MLPs, LayerNorms), adjusting every single weight in the network via the exact same chain rule principles.

Updating the massive $W$ matrices natively takes hundreds of gigabytes of VRAM. To understand how we mathematically bypass this using matrix decomposition, see the dedicated guide: The Mathematics of LoRA & QLoRA.

Key Takeaways

  • Loss isn't just a number: It creates a highly structured gradient vector that describes exactly how wrong every single logit was.
  • Gradients are forces: The gradient acts as an attractive force pulling the correct word vector toward the model's current hidden state, and a repulsive force pushing incorrect words away.
  • Information flows backward: The error from the vocabulary prediction is translated back into the continuous vector space of the model's hidden states, allowing it to correct the millions of parameters that generated the thought.
